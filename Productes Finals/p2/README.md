# P02. Repositori del projecte

# 📂 Organització del Repositori del Projecte

## 🚀 Gestió professional de documentació amb GitHub

Ara ja sabem que GitHub no és només un lloc per pujar fitxers.

És una plataforma molt potent per:

- 📚 documentar projectes,
- 🤝 treballar en equip,
- 🛠️ gestionar configuracions,
- 🧾 mantenir evidències tècniques,
- i centralitzar tota la informació del projecte.

---

# 🎯 Objectiu

En aquest tercer projecte, utilitzareu el repositori creat a la:

## 🧩 Tasca 0

Aquest repositori serà el centre de documentació i lliurament del projecte.

---

# 📋 Organització obligatòria

Per mantenir una estructura professional i coherent, tots els grups hauran de seguir els mateixos criteris.

---

# 🏠 README.md principal

A l’arrel del repositori hi haurà un:

# 📄 `README.md`

Aquest document actuarà com a:

- portada del projecte,
- índex principal,
- punt d’entrada a tota la documentació.

---

## ✅ El README principal ha d’incloure

- descripció del projecte,
- membres del grup,
- estructura del repositori,
- enllaços a les tasques,
- enllaços als productes,
- informació rellevant del projecte.

---

# 📁 Estructura de tasques

Per cada activitat amb lliurament, caldrà crear una carpeta específica.
